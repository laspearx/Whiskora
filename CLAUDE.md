# Whiskora — instructions for Claude

- Read `WHISKORA_PLATFORM_OVERVIEW.md` at repo root before working on anything non-trivial — it's the living reference for what the platform does, the data model, architecture conventions/gotchas, and known gaps.
- After shipping a feature, fixing a bug with structural implications, or changing the data model, **update `WHISKORA_PLATFORM_OVERVIEW.md`**: bump "Last updated", add a line to the Changelog, and touch whichever section changed (feature map, data model, conventions, or known gaps). Keep entries short — one line per item.
